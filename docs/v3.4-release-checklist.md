## Rollback plan

If something breaks post-release:

1. Delete the tag locally and on origin (requires explicit user
   approval; force-delete on a public release is rarely the right call,
   so prefer a v3.4.1 fix forward):

   ```bash
   git tag -d v3.4.0
   git push origin :refs/tags/v3.4.0   # only with explicit approval
   ```

2. Cut a v3.4.1 patch with the fix.

3. Document the issue in the v3.4.1 CHANGELOG entry and link to the
   relevant commit and lesson if one was recorded.
